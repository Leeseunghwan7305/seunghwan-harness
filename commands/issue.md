승환님이 남길 이슈: **$ARGUMENTS**

이 내용을 GitHub 이슈로 만들어줘. 이슈 생성은 **되돌리기 어려운 외부 작업**이니, 실행 전에 반드시 승환님께 확인받아라.

`$ARGUMENTS` 가 비어 있으면 무엇에 대한 이슈인지 먼저 물어보고 멈춰라. (지금 발견한 버그·하려던 작업·리뷰에서 나온 문제 등)

## 사전 점검

1. `gh auth status` 로 gh CLI 로그인 상태를 확인해라. 안 돼 있으면 "`gh auth login` 이 필요합니다" 라고 알리고 멈춰라.
2. `gh repo view` 로 대상 repo 를 확인해라. 원격이 없으면 알리고 멈춰라.
3. `gh issue list --search "<핵심어>"` 로 **유사한 이슈가 이미 있는지** 확인해라. 있으면 새로 만들지 말고 그 이슈를 알려준 뒤, 그래도 새로 만들지 물어봐라.

## 초안 작성 — 생성 전 확인

아래를 **보여주고 승인받은 뒤** 만들어라:

- **제목** — 짧고 구체적으로. 무엇을·어디를. (이슈는 커밋이 아니므로 `feat:` 같은 Conventional Commits 형식은 강제하지 않는다.)
- **본문** — 세 단락:
  - **배경/문제** — 왜 이 이슈인가
  - **맥락·재현** — 관련 파일/경로. 버그면 재현 절차 + 기대 동작 vs 실제 동작
  - **제안/완료 조건** — 무엇이 되면 이 이슈를 닫을 수 있나
- **라벨/담당자** — `$ARGUMENTS` 에 지시가 있으면 반영 (예: `label:bug`, "내 담당으로").

## 생성 — 확인 후

승인하면:

1. `gh issue create --title "..." --body "..."` (라벨 요청 시 `--label <이름>`, 담당 지정 시 `--assignee @me`)
2. 생성된 **이슈 URL** 을 보여줘라.

승환님이 확인하기 전에는 **이슈를 만들지 마라.**
